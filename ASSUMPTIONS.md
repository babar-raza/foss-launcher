# Assumptions

This file documents assumptions made during spec/plan hardening **only when required information could not be definitively derived** from existing documentation.

Assumptions should be:
- **Minimal**
- **Explicit**
- **Risk-labeled**
- **Validated or invalidated** as soon as possible

## Format

Each assumption should include:
- **ID**: Unique identifier (ASM-###)
- **Category**: Area (specs/plans/architecture/requirements)
- **Assumption**: Clear statement of what is assumed
- **Rationale**: Why this assumption was made
- **Risk**: What happens if assumption is wrong
- **Date Added**: When assumption was made
- **Validation Status**: UNVALIDATED | VALIDATED | INVALIDATED
